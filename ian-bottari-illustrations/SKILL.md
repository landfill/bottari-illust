# 여행보따리 캐릭터 본문 삽화

## 핵심 포지션

글을 위한 16:9 가로형 본문 삽화를 설계하고 생성합니다. 목표는 상업 일러스트, PPT 정보 그래픽, 과장된 만화가 아니라 글 안의 핵심 판단, 흐름, 구조, 상태 또는 은유를 산뜻하고, 조용하고, 읽기 쉽지만 설명서처럼 보이지 않는 손그림 설명 이미지로 바꾸는 것입니다.

기본 캐릭터는 "보따리"입니다. 보따리는 작고 둥근 여행용 배낭 캐릭터입니다. 검은 손그림 선으로 앞주머니, 지퍼, 어깨끈, 작은 고리를 간결하게 표현하고, 옆이나 뒤쪽에는 오렌지 점무늬가 있는 작은 보자기 꾸러미를 묶어 둡니다. 얼굴은 작은 눈 두 개와 은은한 미소만 사용합니다. 보따리는 화면의 핵심 동작에 조용히 참여해야 하며, 구석에 세워 두는 장식이면 안 됩니다.

## 먼저 읽을 참고 자료

작업에 필요한 것만 읽고, 한 번에 모든 내용을 맥락에 넣지 마세요.

- `references/style-dna.md`: 스타일 DNA, 색상, 문자, 금지 사항.
- `references/bottari-ip.md`: 보따리 캐릭터의 외형, 성격, 동작 목록, 금지 사항.
- `references/composition-patterns.md`: 구조 유형, 독창적 은유 생성법, 복제 방지 규칙.
- `references/prompt-template.md`: 단일 이미지 생성 프롬프트 템플릿.
- `references/qa-checklist.md`: 생성 후 확인과 반복 수정 규칙.
- `assets/examples/`: 레거시 예시 구도 확인용으로만 사용합니다. 캐릭터 외형 참조는 항상 `references/bottari-ip.md`를 우선합니다.

## 작업 흐름

### 1. 본문 소화

사용자가 제공한 본문, 링크, Notion 페이지, Markdown 파일, 스크린샷 내용을 먼저 읽습니다. 다음을 추려냅니다.

- 핵심 관점이 무엇인지
- 어떤 문단이 인지적 전환을 담당하는지
- 어떤 내용이 그림으로 설명하기 좋은지
- 어떤 지점은 글로 두는 편이 나은지

평균적으로 그림을 배치하지 마세요. "인지 앵커"를 우선 고릅니다. 예: 핵심 판단, 두 개의 끊김점, 입력과 출력의 닫힌 흐름, 분기, 전후 비교, 하나의 소재를 여러 용도로 쓰기, 이어받는 경로, 흔한 함정, 역할 상태 변화.

### 2. 먼저 삽화 전략 출력

사용자가 "어떻게 삽화로 만들지 분석해 줘 / 어떤 지점에 그림이 필요한지 생각해 줘"라고만 요청하면 먼저 샷 목록을 제공합니다. 각 그림에는 다음을 명확히 씁니다.

- 어느 문단 뒤에 넣을지
- 그림의 주제
- 핵심 의미
- 구조 유형
- 보따리가 그림 안에서 무엇을 운반하거나 정리하는지
- 추천 요소
- 추천 한국어 표기어

기본값은 4-8장입니다. 글이 짧으면 1-3장으로 줄이고, 긴 글이어도 쉽게 9장을 넘기지 마세요. 충분하면 멈추고, 본문을 화보처럼 만들지 마세요.

### 3. 단일 이미지 생성

사용자가 "생성 / 출력 / 만들어 줘 / 이미지로 만들어 줘"라고 명확히 요청하면 확인을 기다리지 말고 내장 `image_gen`으로 각 이미지를 따로 생성합니다. 여러 장을 한 이미지에 합치지 마세요.

각 그림은 하나의 핵심 구조만 다룹니다. 프롬프트에는 반드시 다음이 포함되어야 합니다.

- 16:9 가로형 한국어 본문 삽화
- 순백 배경
- 검은색 손그림 선화
- 소량의 오렌지 포인트
- 넉넉한 여백
- 작고 둥근 여행용 배낭 캐릭터 보따리
- 앞주머니, 지퍼, 어깨끈, 작은 고리, 오렌지 점무늬 보자기 꾸러미
- 작은 눈 두 개와 은은한 미소
- 보따리가 핵심 동작의 주체
- PPT, 상업 일러스트, 유치한 어린이 그림, 복잡한 아키텍처, 왼쪽 위 유형 제목 금지

이전 사례를 복제하지 마세요. 기존 예시 자산은 구도 복제 금지 목록을 확인하는 용도에 가깝습니다. "컨베이어 벨트 끊김점 / 레버 당김 / 소재 절단 / 도장 도구함 / 흔한 함정 경로" 같은 기존 구도를 직접 재사용하면 안 됩니다. 사용자가 특정 그림을 복제하라고 명확히 요청한 경우만 예외입니다. 매번 현재 글에서 차분하지만 기억에 남는 새 은유를 발명해야 합니다.

### 4. 확인과 반복 수정

생성 후 `references/qa-checklist.md`를 확인합니다. 다음 문제가 있으면 우선 재생성하거나 부분 편집합니다.

- 보따리가 배낭이 아니라 다른 캐릭터처럼 보임
- 보따리가 장식에 그침
- 앞주머니, 지퍼, 어깨끈, 작은 고리, 보자기 꾸러미 단서가 부족함
- 보자기 꾸러미의 오렌지 포인트가 없거나 너무 많음
- 화면이 너무 가득 참
- 흐름도나 PPT처럼 보임
- 한국어 글자가 너무 많거나 오탈자가 심함
- 왼쪽 위에 "흔한 함정/흐름도/시스템 아키텍처" 같은 제목이 생김
- 그림체가 너무 유치하거나, 상업 마스코트처럼 과장됨
- 배경이 깨끗한 흰색이 아님

### 5. 저장과 전달

사용자가 워크스페이스 안에서 작업 중이면 최종 이미지를 다음 위치에 복사합니다.

```text
assets/<article-slug>-illustrations/
```

순서대로 이름을 붙입니다.

```text
01-topic-name.png
02-topic-name.png
```

사용자가 명확히 교체를 요청하지 않은 한 기존 자산을 덮어쓰지 말고, 원본 생성 파일도 보존합니다.

## 출력 기준

생성 전 전략 출력은 짧고 정확해야 합니다. 생성 후 전달에는 다음을 포함합니다.

- 몇 장을 생성했는지
- 각 그림의 용도
- 저장 경로
- 어떤 그림이 가장 안정적이고, 어떤 그림은 선택 사항인지

스타일 이론을 길게 설명하지 마세요. 그림이 스스로 말하게 하세요.
