# 구도 패턴과 독창성 규칙

## 기본 구조 유형

하나의 구조만 선택하세요. 너무 많이 섞지 마세요.

### 워크플로

적합한 경우: 입력에서 처리로, 처리에서 출력으로 이어지는 내용 생산, AI 작업 흐름, 자동화 연결.

그리는 법: 왼쪽에는 입력, 가운데에는 보따리나 단순한 작업 물건, 오른쪽에는 출력을 둡니다. 흐름은 검은 선과 아주 작은 오렌지 강조점으로 표현합니다.

### 시스템 일부

적합한 경우: 정보 출처, 필터, 데이터베이스, 렌더러, 에이전트 시스템의 일부.

그리는 법: 핵심 모듈 3-5개만 그리고, 보따리가 그중 하나의 중요한 물건을 품거나 다음 칸으로 옮기게 합니다.

### 전후 비교

적합한 경우: 혼란/정돈, 수동/자동, 흩어짐/수렴, 불안/안정.

그리는 법: 왼쪽은 흩어진 종이와 작은 물건, 오른쪽은 묶인 꾸러미나 정리된 주머니. 보따리는 가운데에서 조용히 묶거나 넣거나 내려놓습니다.

### 역할 상태

적합한 경우: 사용자 고통 지점, 창작자 상태, 도구 과다, 정보 불안, 막힘에서 작동으로 바뀌는 과정.

그리는 법: 2-4개의 작은 상태를 그리고, 각 상태마다 짧은 표기를 둡니다. 보따리는 상태마다 과장된 표정 대신 작은 자세 변화만 보입니다.

### 개념 은유

적합한 경우: 콘텐츠 공장, 정보 창고, 머릿속 블랙박스, 작업 흐름 기계, 자동 일일 보고.

그리는 법: 하나의 낮은 기술감 물건이나 생활 도구를 중심에 둡니다. 보따리는 그 물건에 핵심 재료를 넣거나, 꺼내거나, 묶어 갑니다. 기억에 남아야 하지만 포스터처럼 과장하지 않습니다.

### 방법의 층위

적합한 경우: 방법론 프레임워크, 시스템 계층, 역량 스택, 콘텐츠 시스템의 층위.

그리는 법: 여러 층의 상자를 그리되 정식 피라미드처럼 만들지 않습니다. 보따리는 작은 짐을 한 층씩 올려놓거나, 앞주머니에서 라벨을 꺼내 붙입니다.

### 지도 경로

적합한 경우: 아이디어에서 출시까지, 사용자 경로, 콘텐츠 이어받기 경로, 학습 경로.

그리는 법: 굽은 경로 하나와 소수의 노드를 둡니다. 보따리는 길을 천천히 걷거나, 작은 꾸러미를 다음 지점에 내려놓습니다.

### 짧은 만화 컷

적합한 경우: 실패에서 성공으로, 실제 과정, 사용 전후 변화.

그리는 법: 2-4개의 작은 장면으로 구성하고, 각 칸은 하나의 동작만 표현합니다. 과장된 표정 대신 배낭의 위치, 주머니, 꾸러미 상태 변화로 이야기를 만듭니다.

## 독창적 은유 생성법

매번 현재 글에서 은유를 새로 발명하세요. 예전 그림을 옮겨 오면 안 됩니다.

### 세 단계

1. 추상 개념을 물리적 동작으로 바꿉니다. 예: 넣기, 꺼내기, 묶기, 풀기, 접기, 옮기기, 내려놓기, 길 찾기, 표 붙이기, 가볍게 만들기.
2. 시스템 구조를 낮은 기술감의 물건으로 바꿉니다. 예: 종이상자, 서랍, 우체통, 작은 지도, 티켓, 짐 태그, 작업대, 주머니, 실타래, 체크리스트, 수납함, 보자기, 조용한 길.
3. 보따리가 동작을 맡게 합니다. 옆에 서 있는 것이 아니라 앞주머니에 넣고, 지퍼를 열고, 보자기를 묶고, 작은 짐을 옮기고, 길 위에 내려놓고, 다음 사람에게 건네게 합니다.

### 사용할 수 있는 물건 목록

- 종이상자, 서랍, 우체통, 작은 지도, 티켓, 짐 태그, 작업대, 주머니, 실타래, 체크리스트, 수납함, 보자기, 짧은 길, 작은 문, 안내 표식, 가벼운 짐.
- 사용할 때는 1-2개만 고르고, 화면을 가득 채우지 마세요.

### 보따리 동작 목록

- 넣기, 꺼내기, 묶기, 풀기, 메기, 내려놓기, 건네기, 걷기, 기다리기, 연결하기, 접기, 정리하기, 라벨 붙이기, 작은 지도 보기, 티켓 들기, 주머니 잠그기.
- 동작은 핵심 의미를 위해 존재해야 하며, 귀여운 포즈를 만들기 위해 동작을 넣으면 안 됩니다.

## 복제 방지 규칙

기본적으로 `assets/examples/`를 열거나 복제하지 마세요. 현재 폴더의 기존 예시 이미지는 이전 캐릭터 기준으로 만들어진 레거시 자산일 수 있으므로, 보따리 캐릭터의 외형 참조로 사용하지 않습니다. 캐릭터 기준은 항상 `references/bottari-ip.md`가 우선합니다.

사용자가 "이 그림처럼 / 이 구도를 복제 / 이 사례를 바탕으로 수정"이라고 명확히 말하지 않은 한, 다음 기존 구도를 직접 재사용하지 마세요.

- 컨베이어 벨트의 두 끊김점
- 콘텐츠 기계 안에서 캐릭터가 판단 레버를 당김
- 캐릭터가 깔때기가 되어 트래픽/신뢰/전환을 분류함
- 소재를 자르는 구도
- 이어받는 경로를 줄로 끌고 감
- 세 층의 정보원을 줄로 당김
- 세 캐릭터가 각각 확성기/다리 놓기/문 열기를 맡음
- 문구 도구함에 도장을 찍음
- 경고판을 들고 흔한 함정 경로를 바라봄

비슷한 주제라도 새로운 은유로 바꾸세요. 예를 들어 "이어받는 경로"는 꼭 길로 그릴 필요가 없습니다. 보따리가 작은 티켓을 다음 주머니에 넣는 장면일 수도 있습니다. "하나의 소재를 여러 용도로 쓰기"는 보따리가 접힌 종이를 앞주머니, 보자기, 우체통으로 나누어 담는 장면일 수도 있습니다.
