# 프롬프트 예시

아래 프롬프트는 Codex에서 바로 복사해 사용할 수 있습니다.

## 삽화 기획만 하기

```text
$ian-bottari-illustrations로 먼저 이미지는 생성하지 말고 작업해 주세요.
아래 글에서 삽화로 만들 만한 지점을 분석하고, 약 5장의 샷 목록을 출력해 주세요.
각 그림마다 다음을 써 주세요.
- 어느 문단 뒤에 넣을지
- 그림의 주제
- 핵심 의미
- 구조 유형
- 보따리가 그림 안에서 무엇을 운반하거나 정리하는지
- 추천 요소
- 추천 한국어 표기어

<글 붙여넣기>
```

## 글 본문 삽화

```text
$ian-bottari-illustrations로 아래 글을 보따리 배낭 본문 삽화 4장으로 만들어 주세요.
요구사항: 16:9 가로형, 순백 배경, 검은색 손그림 선화, 오렌지 포인트 소량.
보따리는 작고 둥근 여행용 배낭 캐릭터여야 합니다. 앞주머니, 지퍼, 어깨끈, 작은 고리, 오렌지 점무늬 보자기 꾸러미가 보이게 해 주세요.
각 그림은 하나의 핵심 구조만 설명해야 합니다. PPT 정보 그래픽처럼 만들지 말고, 과장된 만화 마스코트처럼도 만들지 마세요.

<글 붙여넣기>
```

## 긴 글 삽화 전략

```text
$ian-bottari-illustrations로 이 긴 글의 삽화 전략을 만들어 주세요.
평균적으로 그림을 배치하지 말고, 핵심 판단, 입력과 출력의 닫힌 흐름, 전후 비교, 흔한 함정, 이어받는 경로 같은 인지 앵커만 고르세요.
기본 6-8장으로 먼저 샷 목록을 출력하고, 이미지는 생성하지 마세요.

<글 붙여넣기>
```

## 단일 관점을 그림 한 장으로 만들기

```text
$ian-bottari-illustrations로 이 관점을 16:9 본문 삽화 한 장으로 만들어 주세요.

신뢰는 외쳐서 생기는 것이 아니라, 증거를 한 장씩 깔아 만든 길이다.

화면은 차분하고 산뜻해야 하고, 보따리가 작은 여행용 배낭처럼 핵심 증거를 조용히 운반해야 합니다.
한국어 표기는 최대 5개, 짧게 써 주세요.
```

## 워크플로 주제

```text
$ian-bottari-illustrations로 "하나의 원본 소재를 트래픽, 신뢰, 전환이라는 세 종류의 콘텐츠로 가공하기"를 그림 한 장으로 만들어 주세요.
정식 흐름도를 그리지 말고, 기존의 하나의 소재를 여러 용도로 쓰는 사례도 복제하지 마세요.
새로운 낮은 기술감의 은유를 다시 발명하고, 보따리가 앞주머니와 보자기 꾸러미를 활용해 핵심 동작에 참여하게 해 주세요.
```

## 이미지 수정: 제목 제거

```text
$ian-bottari-illustrations로 이 그림을 편집해 주세요.
왼쪽 위의 "흐름도" 제목과 밑줄을 제거하고, 다른 내용은 그대로 유지해 주세요.
새 글자나 물건을 추가하지 마세요.
```

## 이미지 수정: 보따리 일관성 강화

```text
$ian-bottari-illustrations로 이 그림의 방향은 맞지만 보따리가 첨부 기준과 다르게 보입니다.
핵심 의미는 유지하되, 보따리를 작고 둥근 여행용 배낭 캐릭터로 다시 맞춰 주세요.
앞주머니, 지퍼, 어깨끈, 작은 고리, 오렌지 점무늬 보자기 꾸러미가 보이게 하고, 표정은 작은 눈 두 개와 은은한 미소만 남겨 주세요.
화면은 순백이고 산뜻하며 글자는 적게 유지해 주세요.
```

## 스타일 샘플 한 묶음 생성

```text
$ian-bottari-illustrations로 서로 다른 주제의 보따리 배낭 본문 삽화 샘플 5개를 출력해 주세요.
주제는 정보 과부하, 제품 검증, 콘텐츠 복리, 1인 회사, 신뢰 구축을 각각 다뤄 주세요.
각 그림은 따로 생성하고, 한 장으로 합치지 마세요.
모든 그림에서 보따리는 작고 둥근 여행용 배낭이며, 오렌지 점무늬 보자기 꾸러미를 소량의 포인트로 포함해야 합니다.
```
